# 账号来源与订阅库联动设计

日期：2026-06-16

## 背景

MakerHub 目前把“线上账号”和“订阅库管理”放在两个入口：

- 设置页的线上账号负责登录辅助、Cookie 保存、账号资料、账号测试和账号来源同步摘要。
- 订阅库负责作者、收藏夹、合集订阅的增删改查、Cron、启停、立即同步和来源卡展示。

这两个入口在数据上已经有关联：保存 Cookie 后，后台会发现关注作者、默认收藏夹和关注合集，并生成订阅来源。近期账号卡已经补充了“订阅库总数、账号来源已同步、其他来源、待解析”等统计口径，但用户仍需要在两个页面之间建立心智连接。

## 决策

不把“线上账号”和“订阅库管理”完全合并成一个页面。保留独立页面职责，并增强两个页面之间的跳转、筛选和归属展示。

理由：

- 线上账号属于 Core / 设置能力，涉及 Cookie、登录、测试和敏感状态。
- 订阅库属于 Subscriptions 能力，涉及来源调度、Cron、启停、同步和删除。
- 合并完整页面会让设置页承载大量订阅 CRUD，增加误删、误停和敏感信息混杂风险。
- 现有模块文档已经把账号和订阅分属不同边界，低风险改法应尊重这个边界。

推荐形态是“强关联、弱合并”：账号页提供来源同步摘要和到订阅库的上下文跳转；订阅库管理页展示来源归属，并支持按账号同步来源筛选。

## 用户体验

### 设置页线上账号

线上账号 tab 继续只承担账号级操作：

- 添加账号。
- 测试 Cookie / 账号状态。
- 删除账号。
- 展示账号来源同步摘要。

每个账号卡增加一个次级操作入口：

- `查看来源`：跳转到订阅库管理页，并带上平台和来源过滤上下文。

账号总览区域保留现有统计口径：

- 订阅库总数。
- 账号来源已同步。
- 其他来源。
- 待解析。

如果账号没有可识别来源，入口仍可显示，但跳转后订阅库管理页应展示空结果和清晰状态，而不是报错。

### 订阅库管理

订阅库管理页继续承担订阅级操作：

- 编辑订阅链接、名称和 Cron。
- 启停订阅。
- 立即同步。
- 删除订阅。
- 订阅设置。

页面增加来源归属信息：

- `账号同步来源`：来自 Cookie 关注作者、默认收藏夹或关注合集。
- `手动添加来源`：用户在订阅库直接添加。
- `其他来源`：无法确认归属或历史迁移来源。

页面增加轻量筛选：

- 全部。
- 账号同步来源。
- 手动添加来源。
- 其他来源。

从账号卡跳转过来时，订阅库管理页默认打开“账号同步来源”筛选，并尽量限定到对应平台；如果无法准确限定到具体账号，只限定平台并保留说明。

### 订阅来源卡

订阅来源卡或管理卡片上增加短标签，用于解释来源归属：

- `来自国区账号 @name`
- `来自国际账号 @name`
- `默认收藏夹`
- `关注作者`
- `关注合集`
- `手动添加`

标签只展示归属，不改变现有卡片排序、同步状态和统计口径。

## 数据与接口

优先复用现有数据：

- `config.cookies`
- `config.subscriptions`
- `cookie_source_inventory`
- `cookie_source_sync_state`
- 当前订阅条目的 `url`、`mode`、`enabled`、`status` 等字段。

第一阶段不新增后端状态表，不改变订阅数据模型，不迁移历史数据。前端可以沿用 `frontend/src/lib/accountSourceStats.js` 的归属匹配思路，通过 URL、平台、作者 handle、作者 uid、收藏夹 id 判断订阅是否来自账号 inventory。

如果现有 API payload 不足以稳定判断来源归属，再考虑后续小步扩展后端 payload，例如为订阅管理列表返回只读的 `source_origin` 字段。该字段只能是派生展示信息，不能成为订阅调度的唯一依据。

## 错误和空状态

- 账号来源 inventory 为空但订阅库存在同平台订阅时，展示为“待解析”或“其他来源”，不要强行归入账号来源。
- 账号同步失败时，账号卡继续显示失败原因；订阅库管理页不应因为同步失败而无法加载。
- 账号已删除后，历史订阅不自动删除，归属展示降级为“其他来源”或“历史账号来源”。
- 删除订阅仍走现有确认逻辑，不因为账号来源标签改变删除语义。

## 测试范围

前端单元测试：

- 账号来源统计继续覆盖账号来源已同步、其他来源、待解析。
- 新增来源归属分类测试，覆盖作者、默认收藏夹、关注合集、手动来源、未知来源。
- 从账号卡生成订阅库管理跳转参数的测试。

前端构建：

```bash
npm --prefix frontend run build
```

如后续扩展后端 payload，再补充对应后端测试：

```bash
.venv/bin/python -m unittest tests.test_subscriptions tests.test_source_library
```

## 非目标

- 不把线上账号完整嵌入订阅库管理页。
- 不把订阅 CRUD 列表完整嵌入设置页。
- 不改变 Cookie 保存、账号登录、订阅同步和 Cron 调度的后端流程。
- 不自动删除账号同步出来的历史订阅。
- 不新增复杂的账号订阅所有权模型。

## 验收标准

- 用户能从线上账号卡直接进入对应账号来源视角的订阅库管理页。
- 用户能在订阅库管理页看出每个订阅大致来自账号同步、手动添加还是其他来源。
- 账号页继续保持设置页的密度和职责，不出现完整订阅 CRUD 列表。
- 订阅库管理页继续保持订阅操作中心定位，不承载账号登录和 Cookie 操作。
- 深色、浅色和自动主题下新增标签、筛选和跳转入口都可读。
